## Implementation notes

- **add-connector** — discovers available connector templates, matches the user's request
  to one, gathers any required credentials/configuration interactively, and installs the
  connector instance on Ingext / Fluency.
- **azure-user-signin-investigation** — runs three saved FPL reports
  (`GetUserSigninHistory`, `GetDirectoryChangesInitiatedByUser`,
  `GetDirectoryChangesTargetingUser`) against the `AzureSigninLogs` / `AzureAuditLogs`
  tables and combines them into one HTML report (sign-in timeline, executive summary,
  per-report tables, recommendations). Requires those FPL reports to be deployed.
- **fluency-report** — lists available reports via `list_reports`, matches the request
  (never invents or substitutes a report), runs it, and renders a single-page,
  Fluency-branded HTML summary using the bundled `assets/base_template.html` and the
  widget recipes in `references/widget_recipes.md`.
- **fortigate-bandwidth** — knowledge skill: the FortiGate byte fields are cumulative
  session counters, so it carries the delta-based aggregation rule, the `logid 0000000020`
  include/exclude decision, and a validation checklist. The runnable query lives in the
  ingext-kql schema KB; this skill is the reasoning, not a duplicate.
- **fpl-report-builder** — authors an FPL report *source file* by compiling one or more
  KQL sections into a single time-bounded report definition. For writing the definition,
  not running it (use fluency-report) and not for single standalone queries (use
  ingext-kql).
- **html-to-pdf** — converts an HTML file to a high-fidelity PDF using TypeScript and
  Playwright (headless Chromium). Commonly chained after a skill that emits HTML.
- **ingext-health-monitor** — checks an Ingext site's health and produces a status report:
  whether data is flowing, router/pipe errors and which pipe is failing, dropped and
  egressed volume, ingestion spikes/outages, and queue backlog.
- **ingext-kql** — discovers tables with the single `list_data_tables` tool and resolves
  field definitions from an embedded schema knowledge base under `references/schemas/`,
  generated from the `ingext_schema` repo by `scripts/sync_schemas.py` (`--check` detects
  drift). Stream tables are time-bounded on `TimeGenerated`; resource snapshot tables are
  queried directly. It never guesses an unknown schema and always parse-validates with
  `validate_kql` before returning.
- **ingext-promql** — generates and runs PromQL / MetricsQL against the platform metrics
  store (VictoriaMetrics) for throughput, ingest/egress volume, component and processor
  rates, and error/drop ratios. Platform metrics only — datalake event search uses
  ingext-kql.
- **office-user-investigation** — runs a fixed KQL set (workload, operations, per-IP
  summary, sign-ins, inbox rules, OAuth consents, daily timeline) directly against the
  `Office365` datalake table, geolocates every source IP offline with GeoLite2, and
  produces a self-contained HTML report with a GeoIP map plus an optional PDF.

### Packaging

The `cowork/*.skill` artifacts have non-uniform internal layouts; the version was injected
into each package's embedded `SKILL.md` in place, except `ingext-kql.skill`, which is
rebuilt from source.
