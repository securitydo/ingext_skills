# 2026-06-23 — azure epoch/time hardening + 7-day default; office-user-investigation drops the GeoIP lookup and pre-bundles all country outlines

Two skills updated. `azure-user-signin-investigation` had its time-range handling reworked to
stop a class of epoch-conversion bug (showing the user one year while passing an epoch from a
different year), and now defaults to the **last 7 days** when no window is supplied instead of
always asking. `office-user-investigation` no longer performs any IP-geolocation lookup — its
per-IP KQL query already returns each source IP's country / city / ISP / coordinates from the
platform `_ip` enrichment, so the bundled offline GeoLite2 path was removed. Its map outlines now
ship **all ~175 sovereign countries** (up from 8), so the build never fetches at runtime, and a
new rule says a country that is somehow still missing must **not** be looked up — estimate it from
known geography instead. For what each skill does and its trigger phrases, see its section in the
root `README.md`.

## Implementation notes

- **azure-user-signin-investigation (1.0.2 → 1.0.3)** — three SKILL.md changes to the
  time-range/argument handling; no query, report-layout, or heuristic changes.
  - **Time-conversion prompt tightened.** The "Required arguments" guidance now instructs the
    skill to convert a chosen preset to milliseconds by **subtracting from the authoritative
    `now` epoch in the CURRENT TIME block of the system prompt** — never computing the current
    epoch from memory — and requires the human-readable date shown to the user to correspond to
    the epoch actually passed. (Fixes the common "shows this year's date, passes last year's
    epoch" bug.)
  - **Epoch bug fix in Step 2.** Step 2 was rewritten with explicit `now_ms`-based presets
    (Last 24 h = `now_ms - 86_400_000`, Last 7 days = `now_ms - 604_800_000`, Last 30 days =
    `now_ms - 2_592_000_000`, `to = now_ms`), using `now_ms` verbatim rather than recomputing it,
    plus a sanity check before running: the year in the label shown to the user must equal the
    year of the epoch passed, otherwise recompute from `now_ms`.
  - **Default 7-day window.** Only `username` is now mandatory. If no time range is supplied, the
    skill defaults to the last 7 days (`from = now_ms - 604_800_000`, `to = now_ms`) and proceeds
    without asking; the 24 h / 30 d / custom presets are still offered on request. The "Required
    arguments" table, the pipeline diagram (step 2), and the Step 2 body were all updated to
    match. This mirrors the default-window behaviour shipped for `office-user-investigation` in
    the 2026-06-22 release.

- **office-user-investigation (1.0.1 → 1.0.2)** — removed the IP-geolocation lookup entirely;
  geo now comes solely from the query. No report layout, KPI, map, or BEC/verdict-heuristic
  changes.
  - **No GeoIP lookup.** The `ip_summary` query already returns `Country`, `CC`, `City`, `ISP`,
    `Lat`, `Lon` per source IP from the platform `_ip` enrichment, so the report has everything it
    needs to place each IP. `build_report.py` now plots `Lat`/`Lon` straight from the query rows.
  - **GeoLite2 path removed.** Deleted `get_geo_reader()` / `geolocate()` and the fallback branch
    that called the bundled offline MaxMind GeoLite2-City DB for un-geolocated IPs. The
    `maxminddb-geolite2` dependency is gone — the first-run install in Step 4 now only installs
    `weasyprint` (PDF rendering). If a tenant's `_ip` enrichment lacks coordinates for an IP, that
    IP is listed without a map dot rather than looked up.
  - **All country outlines pre-bundled.** `assets/countries/` now ships **all ~175 sovereign
    countries** (simplified 110m Natural Earth outlines, `<ISO3>.geo.json`), up from the 8 seeded
    before, so the build never needs to fetch an outline at runtime. Generated offline from the
    `world-atlas` 110m TopoJSON, mapped numeric→ISO3 via `world-countries`, and written in the same
    one-feature `FeatureCollection` (`id`=ISO3, `properties.name`) format the loader already used.
    Kosovo is included as `XKX` and `XK → XKX` was added to `iso2_to_iso3.json`. Step 3, the
    pipeline, the layout, the failure-modes table, the script docstring, and the missing-outline
    `WARN` were updated to say outlines are pre-bundled (a missing one is now only a rare minor
    territory and stays cosmetic — the dot still plots). Verified: the loader parses all 175
    geometries and renders the map SVG without error.
  - **No-lookup rule for missing countries.** Added an explicit note (Step 3 + failure-modes row):
    if a country is somehow not in the bundled set, **do not look up or fetch its outline** — the
    dot already conveys the location, and if a border is worth adding at all, estimate a rough
    outline from your own knowledge and treat it as indicative only.
  - **Docs updated.** Description, intro, the `ip_summary` callout (now v1.0.2), Step 4 notes,
    the script docstring, the layout comment, and the map caveat text all updated to state the geo
    comes from the query and no IP lookup is performed.
  - **Epoch/time-range guidance added.** A new **"Time range → epoch milliseconds"** subsection
    aligns the window handling with `azure-user-signin-investigation`: derive the bounds from the
    authoritative `now_ms` in the system prompt's CURRENT TIME block (used verbatim, never
    recomputed), with presets (24 h / 7 d / 30 d / 90 d) and a year-match sanity check before
    running the queries. The skill's existing "default to last 7 days without asking" behaviour is
    preserved (the sanity check is phrased for this skill's `kql_search` flow, not `run_report`).
  - **Verified**: `build_report.py` compiles and carries no `geolite`/`maxmind`/`reader`
    references; the rebuilt package's embedded script is likewise clean.
